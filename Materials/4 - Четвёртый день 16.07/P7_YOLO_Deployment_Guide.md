# 📝 Руководство к Практике 7: Настройка YOLO на реальном роботе GFS-X

Это руководство описывает процесс переноса и запуска обученного ИИ на реальном гусеничном роботе GFS-X. Скрипт-приемник **`RealVision.cs`** в Unity вам предстоит написать самостоятельно по приведенным ниже шагам.

---

## 🧐 Архитектура HIL (Hardware-in-the-Loop) зрения

Поскольку бортовой компьютер Raspberry Pi 4B слишком слаб для запуска сверточных нейросетей YOLO (выдает всего 1–3 FPS), мы выносим обработку на ПК:

1. **Тело робота (Raspberry Pi)** захватывает видеопоток с камеры и транслирует его в сеть по HTTP на порту `8080`, а также слушает команды ROS.
2. **Мозг робота (Windows PC)** принимает HTTP-видеопоток, прогоняет кадры через YOLO на высокой скорости (30 FPS) в Python-ноде и шлет координаты UDP-пакетами на порт `5005` в Unity.
3. **Unity** принимает координаты в `RealVision.cs`, подает в обученную ONNX-модель и шлет сигналы управления на робота через ROS TCP Endpoint.

### 🔍 Как узнать, какие классы зашиты в модель?

Чтобы узнать, какие именно объекты умеет распознавать загруженная модель и какие ID соответствуют этим классам, вы можете запустить интерактивный сеанс Python (или создать короткий скрипт `check_model.py`) и вывести словарь классов на экран:

```python
from ultralytics import YOLO

# Загружаем модель (работает для .pt, .onnx и OpenVINO папок)
model = YOLO("best_detect.pt")

# Выводим словарь классов в консоль
print("Классы модели:", model.names)
```

Выполнение этого кода для выданных вам весов выведет следующий словарь:
`{0: 'ball', 1: 'cube', 2: 'robot-claw'}`

Это означает:
*   **Класс 0**: `ball` (оранжевый мячик) — цель нашего робота.
*   **Класс 1**: `cube` (куб разметки).
*   **Класс 2**: `robot-claw` (собственная клешня робота — используется для распознавания взаимного положения манипулятора).

Если вы в будущем будете переобучать модель или использовать другую, обязательно проверяйте вывод `model.names`, так как индексы классов могут сдвинуться (например, мяч может стать классом `1` или `2`), что потребует соответствующего изменения параметра `TARGET_CLASSES` в настройках скрипта.

### 📊 Сравнение моделей и рекомендации по использованию

Вам предоставлены две версии архитектуры нейросетей, каждая из которых имеет свои сильные стороны:
*   **YOLOv11** — новейшее семейство моделей. Обладает более высокой точностью локализации объектов и стабильностью распознавания при сложных ракурсах, но требует чуть больше ресурсов.
*   **YOLOv26** — сверхлегкая архитектура, созданная специально для микроконтроллеров и слабых процессоров. Имеет минимальный отклик и низкое энергопотребление.

#### 💡 Рекомендации по выбору формата:
1.  **Если у вас ноутбук с дискретной видеокартой NVIDIA**: Копируйте стандартный файл `best_detect.pt`. PyTorch автоматически задействует CUDA-ядра вашей видеокарты, обеспечивая 60+ FPS без оптимизаций.
2.  **Если у вас процессор Intel (без дискретной видеокартой)**: Используйте формат **OpenVINO** (`openvino26` или `openvino11`). Он компилирует веса под векторные инструкции вашего процессора (AVX-512) и дает максимальную скорость работы.
3.  **Если у вас процессор AMD или Mac M1/M2/M3**: Используйте квантованный формат **ONNX INT8** (`best_int8.onnx`). Это даст прирост скорости на универсальных ядрах CPU.

### 🧠 Образовательный разбор: Архитектура yolo_vision_node.py

Скрипт `yolo_vision_node.py` уже находится в корневой папке вашего репозитория. Вам необходимо разобраться в его внутренних алгоритмах. Ниже подробно описана логика работы его ключевых систем:

#### 1. Многопоточный захват кадров (Борьба с сетевым лагом)
По умолчанию OpenCV считывает кадры из HTTP-стрима в одном потоке с нейросетью. Если время инференса YOLO составляет 35 мс (около 28 FPS), а камера шлет поток с частотой 30 FPS, то в сетевом буфере OpenCV начнет накапливаться «очередь» кадров. Уже через пару минут робот начнет видеть кадры с задержкой в 3-5 секунд.
*   **Решение в коде**: Мы ограничиваем размер системного буфера кадров (`cv2.CAP_PROP_BUFFERSIZE = 1`) и выносим опрос камеры в отдельный фоновый поток:
```python
def capture_thread(cap):
    global latest_frame
    while True:
        ret, frame = cap.read()
        if ret:
            with frame_lock:
                latest_frame = frame  # Перезаписываем единственный буфер свежим кадром
        else:
            time.sleep(0.01)
```
Фоновый поток непрерывно выгребает кадры из сети и оставляет в памяти только **один самый свежий кадр**. Основной поток ИИ берет этот кадр по мере готовности, сводя задержку к нулю.

#### 2. Зачем мы используем Трекер вместо Детектора (`model.track` vs `model.predict`)?
При быстром повороте робота или при засветке камеры солнечным бликом нейросеть YOLO может на 2–3 кадра потерять мяч. При использовании обычного `predict()` робот сразу же «ослепнет», сбросит флаг `seesBall` в `false` и начнет хаотично крутиться на месте.
*   **Решение в коде**: В скрипте вызывается метод `model.track(..., persist=True)`. Он запускает трекер (BoT-SORT с фильтром Калмана). Если YOLO теряет объект из вида, трекер на основе физики движения предсказывает, где мяч должен находиться в следующие доли секунды, сглаживая пропуски кадров.

#### 3. Фильтрация ложных объектов (Выбор по площади)
Если на заднем плане кадра появится другой оранжевый объект (или человек в оранжевой футболке), YOLO найдет два объекта. Чтобы робот не отвлекался, скрипт сравнивает площади bounding box всех найденных целей и выбирает **наибольшую**:
```python
best_box = None
max_area = 0
for box in results[0].boxes:
    x1_, y1_, x2_, y2_ = box.xyxy[0].cpu().numpy()
    area = (x2_ - x1_) * (y2_ - y1_)  # Вычисляем площадь рамки в пикселях
    if area > max_area:
        max_area = area
        best_box = box
```
*Почему это работает*: Мяч, находящийся ближе к роботу, визуально всегда имеет наибольшую площадь рамки на экране.

#### 4. Математика нормализации координат
Для передачи данных в Unity мы переводим пиксели в относительные величины:
*   **Угол**:
    ```python
    x_norm = (center_x - w / 2.0) / (w / 2.0)
    ```
    Делит экран пополам. Точка `0.0` — центр кадра, `-1.0` — левый край, `+1.0` — правый край.
*   **Дистанция**:
    ```python
    y_norm = max(0.0, min(1.0, ball_h / h))
    ```
    Вычисляет отношение высоты рамки мяча к высоте кадра. Чем ближе мяч, тем ближе значение к `1.0`.

#### 5. UDP-клиент для мгновенной связи
Мы пакуем данные в JSON-формат и отправляем по протоколу UDP на локальный порт Unity (`5005`):
```python
data = {
    "angle": float(x_norm),
    "distance": float(y_norm),
    "sees": float(sees),
    "conf": float(conf_val),
    "w": float(ball_w),
    "h": float(ball_h)
}
sock.sendto(json.dumps(data).encode(), (UDP_IP, UDP_PORT))
```
*Почему UDP*: В отличие от TCP, протоколу UDP не требуется подтверждение доставки пакетов. Если один пакет потеряется из-за помех Wi-Fi, нам выгоднее просто дождаться следующего свежего кадра, а не копить очередь на переотправку устаревших данных.

### Шаг 3: Написание C# скрипта RealVision.cs в Unity

Вам нужно написать C# скрипт, который будет слушать UDP-порт `5005` в фоновом потоке, парсить JSON с координатами мяча и передавать их в Unity на главном потоке без зависания графики.

1.  Создайте новый C# скрипт **`RealVision.cs`** в Unity.
2.  Объявите структуру пакета, совпадающую с JSON-форматом Python-ноды:
    ```csharp
    [System.Serializable]
    public class YoloDataPacket
    {
        public float angle;      // Отклонение центра мяча (-1.0 лево, 1.0 право)
        public float distance;   // Высота рамки мяча относительно кадра (0..1)
        public float sees;       // Флаг видимости (1.0 = виден, 0.0 = нет)
        public float conf;       // Уверенность детекции
        public float w;          // Ширина bounding box
        public float h;          // Высота bounding box
    }
    ```
3.  Реализуйте фоновое чтение сокета с использованием `ConcurrentQueue` (потокобезопасной очереди) для передачи данных на главный поток Unity:
    ```csharp
    using UnityEngine;
    using System;
    using System.Threading;
    using System.Threading.Tasks;
    using System.Net.Sockets;
    using System.Collections.Concurrent;

    public class RealVision : MonoBehaviour
    {
        public int udpPort = 5005;
        public bool useYOLO = false;
        
        // Телеметрия для RobotBrain
        public float normalizedAngle;
        public float normalizedDistance;
        public bool seesBall;

        private CancellationTokenSource cts;
        private ConcurrentQueue<YoloDataPacket> udpQueue = new ConcurrentQueue<YoloDataPacket>();

        void Start()
        {
            cts = new CancellationTokenSource();
            // Запуск UDP-слушателя в фоновом потоке, чтобы не вешать игру
            Task.Run(() => UdpListenerLoop(cts.Token));
        }

        private async Task UdpListenerLoop(CancellationToken token)
        {
            using (var udpClient = new UdpClient(udpPort))
            {
                while (!token.IsCancellationRequested)
                {
                    var result = await udpClient.ReceiveAsync();
                    string json = System.Text.Encoding.UTF8.GetString(result.Buffer);
                    
                    YoloDataPacket packet = JsonUtility.FromJson<YoloDataPacket>(json);
                    if (packet != null)
                    {
                        udpQueue.Enqueue(packet); // Безопасно кладем в очередь
                    }
                }
            }
        }

        void Update()
        {
            // Читаем пакеты из очереди на главном потоке Unity
            while (udpQueue.TryDequeue(out var packet))
            {
                useYOLO = true;
                seesBall = packet.sees > 0.5f;

                if (seesBall)
                {
                    // Ограничиваем угол [-1, 1]
                    normalizedAngle = Mathf.Clamp(packet.angle, -1f, 1f);
                    
                    // Записываем нормализованную высоту рамки (чем больше мяч, тем он ближе)
                    normalizedDistance = packet.distance; 
                }
                else
                {
                    normalizedAngle = 0f;
                    normalizedDistance = 1f; // 1.0 = далеко/мяч не виден
                }
            }
        }

        void OnDestroy()
        {
            cts?.Cancel(); // Останавливаем фоновый поток при выходе
        }
    }
    ```

# 🤖 Инструкция по подключению к роботу и настройке параметров

Данный гайд содержит информацию о том, как подключиться к роботу **XiaoRGeek GFS-X** по SSH, запустить его системы и какие параметры нужно настраивать для калибровки движения и манипулятора.

---

## 1. Подключение по SSH
Для подключения к роботу откройте терминал (PowerShell на Windows, Terminal на macOS/Linux) и введите команду:

```bash
ssh pi@192.168.2.xx
```
* Где `xx` — это номер вашего робота (написан на корпусе). Например, для робота №160 команда будет:
  `ssh pi@192.168.2.160`
* **Пароль:** `raspberry`

---

## 2. Переход в папку вашей команды
На роботе настроены изолированные окружения для каждой команды, чтобы вы не перезаписывали настройки друг друга. После входа введите:

```bash
cd team1
# или
cd team2
```

---

## 3. Два главных скрипта запуска (.sh)
В папке вашей команды находятся два скрипта запуска с расширением `.sh`:

### 🚀 Скрипт 1: Запуск ROS и моста с Unity
```bash
./start_robot_team1.sh
# или
./start_robot_team2.sh
```
* **Как работает:** 
  1. Очищает старый Docker-контейнер `xiao_ros_brain`.
  2. Запускает чистый контейнер на базе ROS Noetic.
  3. Копирует ваши файлы логики (`unity_master_teamX.py`, `unity_gripper_ir_teamX.py` и др.) внутрь контейнера.
  4. Запускает `roscore` и ROS-TCP Endpoint (порт `10000`) для соединения с Unity.
  5. Запускает основные Python-ноды управления в фоновом режиме.

### 📹 Скрипт 2: Запуск трансляции камеры (MJPEG)
```bash
./camera_stream_team1.sh
# или
./camera_stream_team2.sh
```
* **Как работает:** Запускает на хост-системе Raspberry Pi (вне Docker) HTTP-сервер трансляции видеопотока с камеры на порту **`8080`**.
* **Как проверить:** Откройте браузер на вашем ПК и перейдите по адресу `http://192.168.2.xx:8080/`. Вы должны увидеть живое видео с камеры робота. Это видео считывается YOLO-моделью на ПК.

---

## 4. Что менять и как это работает

Основные файлы логики написаны на Python. Вы можете редактировать их прямо на роботе с помощью редактора `nano` (например, `nano unity_master_team1.py`) либо изменять их на ПК и копировать на робота через `scp` / FileZilla.

### 📄 Файл `unity_master_teamX.py` (Основной мост управления)
Этот файл принимает команды от Unity (`/cmd_vel`, `/cmd_gripper`, `/cmd_camera_pan`), управляет моторами и сервоприводами, а также отправляет данные датчиков обратно в Unity (`/sensor/data`).

Основные параметры для калибровки и настройки робота:

#### ⚙️ Настройка моторов и плавности движения
* **`L`** (строка 56): Колесная база робота (расстояние между гусеницами в метрах). Влияет на расчет поворотов.
* **`TURN_K`** (строка 178): Коэффициент чувствительности поворота. Если робот крутится слишком резко и теряет мяч из виду, **уменьшите** его (например, до `0.20` или `0.25`).
* **`MAX_LINEAR`** (строка 179): Ограничение максимальной линейной скорости (м/с). Рекомендуется установить в `0.25` для безопасного тестирования в помещении.
* **`MIN_MOTOR_PWM`** (строка 59): Минимальная мощность (ШИМ в %), при которой моторы начинают преодолевать трение покоя (обычно от `30` до `40`).
* **`MOTOR_DEAD_ZONE`** (строка 113): Мертвая зона ШИМ. Всё, что ниже этого значения (например, `< 10%`), принудительно сбрасывается в `0`, чтобы моторы не пищали в простое.
* **`MAX_PWM_STEP`** (строка 63): Шаг плавного разгона (Soft-start). Ограничивает изменение ШИМ за один шаг (20мс). Помогает сберечь редукторы и электронику от бросков пускового тока при резкой смене направления.

#### 🦾 Настройка манипулятора (сервоприводы)
* **`ANGLE_CLAW_OPEN`** / **`ANGLE_CLAW_CLOSE`** (строки 223–224): Углы сервопривода клешни (№4) для открытого и закрытого состояний. **Внимание:** Не устанавливайте слишком большие углы закрытия (например, `> 95`), чтобы сервопривод не заклинило и он не сгорел от перегрузки!
* **`ANGLE_SHOULDER_DOWN`** / **`ANGLE_ELBOW_DOWN`** (строки 221–222): Углы опускания манипулятора (плечо №2 и локоть №3) перед захватом мяча. Настройте их так, чтобы клешня находилась параллельно полу на высоте мяча.
* **`ANGLE_SHOULDER_UP`** / **`ANGLE_ELBOW_UP`** (строки 219–220): Углы подъема манипулятора в транспортное состояние после захвата.

---

### 📄 Файл `unity_gripper_ir_teamX.py` (Автоматический захват)
Этот файл опрашивает ИК-датчик наличия мяча в клешне (`IR_M` на пине `22`) и отправляет команду на закрытие.

* **`POLL_HZ`** (строка 50): Частота опроса ИК-датчика (Гц). По умолчанию `20 Hz`.
* **`DEBOUNCE_COUNT`** (строка 51): Фильтр дребезга контактов. Указывает, сколько раз подряд датчик должен стабильно выдать сигнал преграды, чтобы сработал захват. Помогает избежать ложных закрытий при случайных помехах.

---

## 5. Порядок работы и перезапуск

1. Подключитесь по SSH и перейдите в папку вашей команды.
2. Отредактируйте файлы скриптов (например, `nano unity_master_team1.py`).
3. Сохраните изменения (`Ctrl+O`, затем `Enter`, выход — `Ctrl+X`).
4. **Перезапустите робота**, чтобы применить изменения внутри Docker-контейнера:
   ```bash
   ./start_robot_team1.sh
   ```
5. **Проверьте логи на ошибки:**
   * Логи главного моста:
     ```bash
     docker exec xiao_ros_brain cat /tmp/master.log
     ```
   * Логи автозахвата клешни:
     ```bash
     docker exec xiao_ros_brain cat /tmp/gripper_ir.log
     ```

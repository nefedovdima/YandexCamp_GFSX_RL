### Обзор сервисов

Для начала работы с роботом достаточно открыть его IP в браузере. На домашней странице находятся ссылки на основные сервисы для взаимодействия с роботом:

- Foxglove &ndash; инструмент визуализации и управления ROS-системой. Ссылка содержит параметры для автоматического подключения к роботу
- Jupyter Lab &ndash; веб-редактор кода, запущенный внутри docker-контейнера
- Terminal (host) &ndash; доступ к терминалу на raspberry через браузер. Здесь можно запустить команды, недоступные из контейнера
- Terminal (docker) &ndash; доступ к терминалу (tmux-сессии) внутри контейнера. Здесь можно посмотреть логи основного ROS-пайплайна или перезапустить его
- System Tests &ndash; сервис для тестирования и диагностики робота
- Camera Stream &ndash; стрим с камеры робота в реальном времени

### Foxglove

Foxglove — это web-приложение для визуализации и отладки ROS проектов, полностью заменяющее более старую технологию RViz. Основным неудобством RViz является необходимость установки его как графического приложения на компьютер пользователя, в то время как foxglove можно просто открыть в браузере.

Краткий обзор возможностей foxglove:

- Подключаться к ROS-пайплайну на роботе или в docker-контейнере ("live connection")
- Воспроизводить записанные данные (mcap файлы, записанные через rosbag record)
- Читать и публиковать сообщения в/из ROS-топиков, вызывать ROS-сервисы и т.п.
- Визуализировать пространственные данные (дерево трансформаций, лидарные сканы и облака точек, карты и costmap, траектории и маркеры...)
- Визуализировать структуру и положение частей робота (URDF, joint states...)
- Показывать граф связности нод и топиков (аналог rqt_graph)

#### Разновидности foxglove

Изначально foxglove был open-source проектом, но через некоторое время стал закрытым и требующим авторизации в сервисе и платной подписки для использования дополнительных функций. Наша лаборатория использует последнюю доступную open-source web-версию foxglove, в которую внесен ряд доработок. Web-версия доступна по двум адресам:

- [foxglove.robotics-lab.ru](https://foxglove.robotics-lab.ru) — HTTP версия сайта
- [foxglove-ssl.robotics-lab.ru](https://foxglove-ssl.robotics-lab.ru) — HTTPS версия сайта

Проще всего использовать HTTP версию, т.к. для соединения с роботом или docker-контейнером используется незащищенное websocket-соединение. При первом посещении HTTP версии сайта браузер может показывать предупреждение "это соединение небезопасно" или похожее — это нормально, нужно просто его пропустить (нажать "все равно перейти к сайту" или похожую кнопку).

При работе по HTTPS протоколу требуется дополнительное разрешение от браузера, чтобы HTTPS (защищенная) страница могла обращаться к HTTP (незащищенному) websocket-соединению, а некоторые браузеры совсем блокируют подобный функционал. Foxglove умеет определять подобные ситуации и предлагает выдать соответствующее разрешение, если это возможно.

Также есть возможность (но не рекомендуется) пользоваться foxglove через официальный сайт [app.foxglove.dev](https://app.foxglove.dev/) или даже [установить его](https://foxglove.dev/download) как standalone-приложение (при этом потеряв весь смысл задумки "тонкого браузерного клиента" и преимущества относительно RViz). В этом случае необходима регистрация аккаунта.

#### Основная область

![](images/foxglove-4.png)

Основная (правая) область состоит из панелей — мини-окошек, отображающих информацию в разном виде в зависимости от типа панели. Практически все панели имеют в верхнем правом углу два индикатора: меню настроек [1] и меню действий [2]. Меню настроек открывается в левой части экрана и отличается в зависимости от типа панели. Есть панели, где  настройки задаются прямо в основной области, например в панели Raw Messages, отображающие сообщения из топика в "сыром виде" (топик задается прямо в верхней части панели). Меню действий [2] всегда одинаковое и содержит базовые действия с панелью:

- Change panel [3] — заменить тип панели на другой без изменения размера и расположения. Настройки панели при этом не сохраняются
- Split horizontal/vertical [4] — "раздвоить" панель по вертикали или горизонтали, с сохранением всех настроек
- Fullscreen [5] — временно вывести панель на весь экран
- Remove panel [6] — удалить панель

Размер панелей можно менять, потянув за границы между ними, а также перемещать (кликнув и не отпуская перетаскивая за верхнюю полоску) и "стакать" с другими по вертикали или горизонтали. Проще показать это на видео:

![](images/foxglove-5.webm)

На видео не видно курсора, поэтому еще раз — перетаскиваются панели за верхнюю полоску, а в случае если там кликабельное поле (как в Raw Messages), то за кнопку меню действий

![](images/foxglove-6.png)

### Заливка и настройка образа

Настройка SD карты для робота происходит в несколько этапов:

1. Заливка официального образа Raspberry Pi OS на SD карту
2. Запуск Raspberry Pi с этой SD картой
3. Запуск скрипта setup.sh для финальной настройки системы
4. Запуск системных тестов робота (опционально)

Вам понадобится:

1. Качественная Micro SD карта, рекомендуемый объем 64GB. Лучше выбирать известных производителей (SanDisk, Samsung, Kingston) и ориентироваться на метки "A1" или "A2" (performance class) и "UHS-I" или "I" (bus speed).
1. Внешний USB кард-ридер для SD (Micro SD) карт. Крайне рекомендуется кард-ридер с поддержкой USB 3.0 (например Transcend RDF5), это сильно влияет на скорость заливки образа. Кард-ридеры, встроенные в ноутбуки или USB-хабы, зачастую медленные.
1. Программа [Raspberry Pi Imager](https://www.raspberrypi.com/software/) для заливки образов
1. Робот или отдельный Raspberry Pi 5 для выполнения 2 и 3 этапов выше

#### Заливка официального образа

Запустите Raspberry Pi Imager и последовательно выберите следующие настройки:

- Device type: Raspberry Pi 5
- OS: Raspberry Pi OS Lite (64-bit)
- Storage: ваш подключенный кард-ридер
- Раздел customization:
- Hostname: rbm-(номер робота), например rbm-42
- Localization:
  - Capital city: Moscow (Russia)
  - Time zone: Europe/Moscow
  - Keyboard layout: ru
- User:
  - Username: `robomarvel`
  - Password: `robomarvel`
- Wi-Fi:
  - Secure Network
  - SSID: `robomarvel`
  - Password: `robomarvel`
- Remote access:
  - Enable SSD: включено
  - Authentication mechanism: Use password authentication
- Raspberry Pi Connect &ndash; не включать

В разделе Writing подтвердите заливку образа и дождитесь окончания процесса. На быстром кард-ридере это занимает около 3-5 минут.

⚠️ Если вы заливаете несколько SD карт последовательно, рекомендуется перезапускать Raspberry Pi Imager, потому что настройки кастомизации (например хостнейм) могут применяться некорректно

#### Настройка системы

После заливки образа на SD карту, вставьте ее в Raspberry Pi и включите питание. Через небольшое время (меньше минуты) Raspberry выполнит автоматическую перезагрузку для начальной настройки (изменение размера файловой системы под размер SD карты и применение пользовательских настроек, выбранных в Raspberry Pi Imager). Заметить перезагрузку можно по кратковременному выключению лидара сверху робота, или по индикатору с задней стороны Raspberry.

После перезагрузки убедитесь, что робот находится в зоне действия стандартной wifi-сети `robomarvel` и появился в списке подключенных устройств (см. [Определение IP робота](#Определение-IP-робота)). Подключитесь к роботу по SSH и запустите команду:

```
curl --create-dirs -Lo ~/.setup/setup.sh "https://raw.githubusercontent.com/AndBondStyle/rbm/refs/heads/master/scripts/setup.sh" && bash ~/.setup/setup.sh
```

Команда запустит новую tmux-сессию и начнет настройку системы. После завершения скрипта система автоматически перезагрузится, если это было необходимо. Эту команду также можно запускать на роботах, которые уже были настроены ранее (т.е. запускать повторно) &ndash; в этом случае скрипт обновит необходимые компоненты (сервисы, репозиторий, контейнер) до последних доступных версий.

### Определение IP робота

По умолчанию робот подключается к wifi сети `robomarvel` с паролем `robomarvel`. Для определения IP робота предполагается, что ваш компьютер также подключен к этой сети. Для определения IP робота (или роботов) в сети есть несколько вариантов:

#### Через веб-интерфейс роутера, если к нему есть доступ

Обычно этот интерфейс доступен по адресу [192.168.0.1](http://192.168.0.1) или [192.168.1.1](http://192.168.1.1), а логин/пароль для входа указаны на этикетке с обратной стороны роутера. Внешний вид интерфейса может отличаться в зависимости от производителя, но обычно имеет раздел с названием, похожим на "Список подключенных устройств". В этом разделе можно увидеть таблицу хостнеймов и IP-адресов устройств, включая вашего робота (хостнейм rbm-\*).

#### Через специализированные приложения

Существует множество приложений, реализующих функционал сканирования локальной сети, например [Angry IP Scanner](https://angryip.org/download/) (windows/mac/linux) или [Fing](https://www.fing.com/) (windows/mac/linux + ios/android).

#### Через утилиту nmap (в командной строке)

Поставить утилиту nmap можно с [официального сайта](https://nmap.org/) или через менеджер пакетов вашей ОС.

Для сканирования сети вам нужно знать IP адрес вашего компьютера. Обычно его можно определить командой `ifconfig` или `ipconfig`, или найти в разделе сетевых настроек системы. Для примера предположим, что IP нашего копьютера `192.168.1.123`. Для выполнения сканирования запустите команду ниже, исправив IP на адрес вашего компьютера:

```
nmap -sn -oG - 192.168.1.123/24
```

Результат выполнения будет содержать хостнеймы и адреса найденных устройств:

```
# Nmap 7.98 scan initiated [...]
Host: 10.1.30.1 ()	Status: Up
Host: 10.1.30.57 (abs-framework.local)	Status: Up
Host: 10.1.30.69 ()	Status: Up
Host: 10.1.30.86 (rbm-123.local)	Status: Up
# Nmap done [...] 256 IP addresses (4 hosts up) scanned
```

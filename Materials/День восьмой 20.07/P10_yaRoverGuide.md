# Практический гайд по Яндекс.Роверу: Доехать до точки и передать цель в Unity

Данный документ содержит полное пошаговое руководство по работе с Яндекс.Ровером на практическом занятии. 

В гайде подробно объясняется смысл каждого шага, устройство главного веб-меню Ровера, способ получения токенов Yandex Cloud, а также построчный разбор Python-скрипта диспетчеризации.

---

## 1. Смысл и цель работы (Зачем мы это делаем?)

На практическом занятии студенты строят первую часть **автономной эстафеты M2M (Machine-to-Machine)**:

1. **Робот-перехватчик GFS-X** на старте "слеп" к дальним объектам — у него нет 3D-лидара и карты лабиринта. Он находится в режиме ожидания (`IDLE`) и не знает, какой именно предмет нужно будет захватить.
2. **Яндекс.Ровер (Скаут)** выполняет навигацию: доезжает по карте через лабиринт к зоне с предметами и фиксирует кадр с видеокамеры.
3. **Модель Yandex GPT (VLM)** анализирует кадр с Ровера по системному промпту и формирует две JSON-команды:
   * **JSON Команда 1 (`set_target_class`)**: Устанавливает Target Class системы зрения YOLO для робота GFS-X. Без этого GFS-X не узнает, какой именно объект искать на полу.
   * **JSON Команда 2 (`activate`)**: Передает команду активации робота GFS-X.
4. **Unity Mission Control** принимает эти команды по сети. Робот GFS-X "просыпается", распознает предмет по обновленному классу YOLO, подезжает к нему и берет клешнёй.

## 2. Главное веб-меню Ровера

При включении Ровера на нем автоматически запускаются все необходимые службы. 

Для удобного доступа к управлению откройте **Яндекс Браузер** на своем ноутбуке и введите в адресную строку IP-адрес Ровера:
```text
http://<IP_ВАШЕГО_РОВЕРА>
```

Перед вами откроется **главное меню Ровера**, где в один клик доступны все интерфейсы:
* **Foxglove Studio** (порт 8765) — для управления навигацией и просмотра карты.
* **Jupyter Lab** (порт 8080) — для написания и запуска Python-скриптов.
* **MediaMTX Video Stream** (порт 8889) — для просмотра видеопотока с камеры.
* **Документация и статусы служб**.

## 3. Где взять IAM-токен и Folder ID в Yandex Cloud

Для отправки запросов в VLM (Yandex GPT API) необходимы два параметра авторизации:

1. **Folder ID (Идентификатор каталога)**:
   * Зайдите в консоль управления Yandex Cloud (`console.yandex.cloud`).
   * Выберите рабочий каталог проекта.
   * В верхней части страницы или в настройках каталога скопируйте значение `Folder ID` (строка формата `b1g...`).

2. **IAM-токен (Временный ключ доступа)**:
   * IAM-токен действует до 12 часов.
   * Если у вас установлен Yandex Cloud CLI, выполните в терминале команду:
     ```bash
     yc iam create-token
     ```
   * Либо скопируйте готовый IAM-токен из списка, выданного организаторами Кемпа.

## 4. Пошаговый порядок выполнения практики

### Шаг 1. Открытие Foxglove Studio из веб-меню
1. В Яндекс Браузере перейдите по адресу `http://<IP_ВАШЕГО_РОВЕРА>`.
2. В появившемся меню кликните по ссылке **Foxglove Studio** (или откройте установленное приложение Foxglove).
3. При запросе адреса подключения укажите: `ws://<IP_ВАШЕГО_РОВЕРА>:8765` и нажмите **Connect**.
4. Проверьте, что на экране отображаются карта помещений (`/map`), скан лидара (`/scan`) и видеопоток.

### Шаг 2. Установка цели на карте и доезд
1. В верхнем меню Foxglove Studio выберите инструмент **2D Goal Pose**.
2. Кликните по карте перед зоной с объектами, указав желаемое направление Ровера.
3. Яндекс.Ровер автоматически построит маршрут по Nav2, объедет коробки и остановится перед объектом.
4. Проверьте по видеопотоку, что целевой объект находится в кадре.

### Шаг 3. Создание Python-скрипта в Jupyter Lab
1. Вернитесь во вкладку главного веб-меню `http://<IP_ВАШЕГО_РОВЕРА>` и кликните на **Jupyter Lab** (или откройте `http://<IP_ВАШЕГО_РОВЕРА>:8080`).
2. В файловом менеджере Jupyter нажмите кнопку **+** (Launcher) и создайте новый файл Python (`rover_dispatcher.py`) или файл ноутбука (`.ipynb`).
3. Скопируйте и напишите код из раздела 6.
4. Вставьте свои значения: `<IP_ВАШЕГО_РОВЕРА>`, `<IP_ПК_С_UNITY>`, `IAM_TOKEN` и `FOLDER_ID`.
5. Запустите скрипт на выполнение.

## 5. Подробный разбор функций Python-скрипта

Скрипт состоит из 4 логических функций, обеспечивающих работу M2M-цепочки:

1. **Функция `capture_rover_frame()` (Получение снимка с камеры)**:
   * **Зачем нужна**: Ровер должен "увидеть" сцену перед собой.
   * **Как работает**: Подключается через библиотеку OpenCV (`cv2.VideoCapture`) к HTTP-видеопотоку MediaMTX (`http://<IP_ВАШЕГО_РОВЕРА>:8889/camera`). Считывает один кадр, кодирует его в кодировку Base64 для возможности передачи текстовым HTTP POST-запросом.

2. **Функция `query_yandex_vlm()` (Анализ снимка нейросетью VLM)**:
   * **Зачем нужна**: Передать изображение в облачный искусственный интеллект для определения предмета.
   * **Как работает**: Формирует JSON-запрос к Yandex GPT API. В запрос передается `system_prompt`, который жестко обязывает нейросеть не писать вводных слов, а вернуть строго JSON-массив из двух элементов.

3. **Функция `parse_json_commands()` (Распаковка команд от ИИ)**:
   * **Зачем нужна**: Извлечь чистые JSON-данные из текстового ответа нейросети.
   * **Как работает**: Удаляет возможные теги разметки (```json) и с помощью `json.loads()` преобразует текст в списки Python.

4. **Функция `send_to_unity_mission_control()` (Передача в Unity)**:
   * **Зачем нужна**: Передать управляющие команды на компьютер оператора с Unity для старта второго робота.
   * **Как работает**: Выполняет два сетевых HTTP POST запроса (`requests.post`):
     * Первый запрос (`/set_target`) передает JSON 1 и устанавливает Target Class системы зрения YOLO на роботе GFS-X.
     * Второй запрос (`/activate`) передает JSON 2 и переключает статус робота GFS-X в состояние старта.

## 6. Исходный код Python-скрипта диспетчеризации

Ниже представлен полный шаблон кода с подробными комментариями:

```python
import base64
import json
import requests
import cv2

# =====================================================================
# 1. КОНФИГУРАЦИЯ ПАРАМЕТРОВ И СЕТЕВЫХ АДРЕСОВ
# =====================================================================
# Укажите IP-адрес вашего Ровера и IP-адрес ПК с Unity Mission Control
ROVER_IP = "<IP_ВАШЕГО_РОВЕРА>"
UNITY_IP = "<IP_ПК_С_UNITY>"

# Адреса видеопотока Ровера и REST API сервера Unity
ROVER_CAM_URL = f"http://{ROVER_IP}:8889/camera"
UNITY_API_URL = f"http://{UNITY_IP}:8088/api/m2m"

# Авторизация в Yandex Cloud (VLM)
YANDEX_GPT_URL = "https://llm.api.cloud.yandex.net/foundationModels/v1/completion"
IAM_TOKEN = "ВАШ_IAM_TOKEN"
FOLDER_ID = "ВАШ_FOLDER_ID"


# =====================================================================
# 2. ФУНКЦИЯ ЗАХВАТА КАДРА С КАМЕРЫ РОВЕРА
# =====================================================================
def capture_rover_frame():
    """Подключается к видеопотоку MediaMTX и возвращает кадр в формате Base64."""
    print("1. Подключение к видеопотоку Ровера...")
    cap = cv2.VideoCapture(ROVER_CAM_URL)
    ret, frame = cap.read()
    cap.release()

    if not ret:
        raise RuntimeError(f"Не удалось получить кадр по адресу {ROVER_CAM_URL}")

    # Сжатие кадра в JPG и кодирование в Base64
    _, buffer = cv2.imencode('.jpg', frame)
    base64_image = base64.b64encode(buffer).decode('utf-8')
    print("Кадр успешно захвачен и закодирован.")
    return base64_image


# =====================================================================
# 3. ФУНКЦИЯ ЗАПРОСА К YANDEX GPT (VLM API)
# =====================================================================
def query_yandex_vlm(base64_image):
    """Отправляет кадр в Yandex GPT VLM и получает 2 JSON-команды."""
    print("2. Отправка снимка в Yandex GPT (VLM)...")
    
    # Системный промпт жестко задает формат вывода ответа
    system_prompt = (
        'Ты VLM-диспетчер M2M эстафеты. Проанализируй изображение. '
        'Ответь строго JSON-массивом из 2 элементов без дополнительного текста: '
        '[{"target":"gfsx_yolo","action":"set_target_class","class_name":"<название_объекта>"},'
        '{"target":"gfsx_robot","action":"activate","state":"start"}]'
    )

    payload = {
        "modelUri": f"gpt://{FOLDER_ID}/yandexgpt/latest",
        "completionOptions": {"temperature": 0.1, "maxTokens": 500},
        "messages": [
            {"role": "system", "text": system_prompt},
            {"role": "user", "text": f"data:image/jpeg;base64,{base64_image}"}
        ]
    }

    headers = {
        "Authorization": f"Bearer {IAM_TOKEN}",
        "x-folder-id": FOLDER_ID,
        "Content-Type": "application/json"
    }

    response = requests.post(YANDEX_GPT_URL, headers=headers, json=payload)
    if response.status_code != 200:
        raise RuntimeError(f"Ошибка Yandex GPT API: {response.status_code} - {response.text}")

    result_text = response.json()['result']['alternatives'][0]['message']['text']
    
    # Очистка ответа от возможной markdown-разметки ```json ... ```
    clean_json_text = result_text.replace("```json", "").replace("```", "").strip()
    commands = json.loads(clean_json_text)
    
    print("Успешно получены команды от VLM:")
    print(json.dumps(commands, indent=2, ensure_ascii=False))
    return commands


# =====================================================================
# 4. ФУНКЦИЯ ПЕРЕДАЧИ КОМАНД В UNITY MISSION CONTROL
# =====================================================================
def send_to_unity_mission_control(commands):
    """Отправляет полученные JSON-команды на компьютер с Unity."""
    print("3. Передача команд в Unity Mission Control...")

    # Отправка JSON 1: Устанавливает Target Class системы зрения YOLO на GFS-X
    res1 = requests.post(f"{UNITY_API_URL}/set_target", json=commands[0])
    print(f"JSON 1 (Target Class) отправлен. Статус ответа Unity: {res1.status_code}")

    # Отправка JSON 2: Активация робота GFS-X
    res2 = requests.post(f"{UNITY_API_URL}/activate", json=commands[1])
    print(f"JSON 2 (Активация GFS-X) отправлен. Статус ответа Unity: {res2.status_code}")

    print("✅ M2M-цепочка успешно завершена!")


# =====================================================================
# 5. ТОЧКА ВХОДА СКРИПТА
# =====================================================================
if __name__ == '__main__':
    try:
        # Запуск цепочки
        img_b64 = capture_rover_frame()
        json_cmd = query_yandex_vlm(img_b64)
        send_to_unity_mission_control(json_cmd)
    except Exception as e:
        print(f"❌ Произошла ошибка при выполнении скрипта: {e}")
```

## 7. Возможные проблемы и быстрые решения

1. **Главная веб-страница `http://<IP_ВАШЕГО_РОВЕРА>` не открывается**:
   * Убедитесь, что ваш ноутбук подключен к той же Wi-Fi сети, что и Ровер.
   * Откройте страницу через **Яндекс Браузер**.

2. **В Foxglove Studio не отображается карта**:
   * Проверьте, что адрес WebSocket указан верно: `ws://<IP_ВАШЕГО_РОВЕРА>:8765`.

3. **Ошибка 401 в Yandex GPT API**:
   * Истёк срок действия `IAM_TOKEN`. Сгенерируйте новый токен через `yc iam create-token`.

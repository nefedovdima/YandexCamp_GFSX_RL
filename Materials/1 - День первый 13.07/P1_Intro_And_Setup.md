## Часть 1: Системное ПО и Python-окружение

### Шаг 1: Установка Git
1. Перейдите на [официальный сайт Git](https://git-scm.com/).
2. Скачайте и запустите установщик для вашей ОС (Windows/Linux/macOS).
3. **Важно (для Windows)**: При выборе компонентов обязательно активируйте опцию **"Git from the command line and also from 3rd-party software"**. Остальные параметры оставьте по умолчанию.

> **Критерий самоконтроля:**
> Откройте командную строку (cmd) или PowerShell и введите:
> ```bash
> git --version
> ```
> *Ожидаемый результат:* Вывод версии программы (например, `git version 2.x.x`).

---

### Шаг 2: Установка Miniconda
1. Перейдите на [официальный сайт Miniconda](https://docs.anaconda.com/miniconda/).
2. Скачайте установщик для вашей операционной системы.
3. Запустите установку с параметрами по умолчанию.
4. **Важно (для Windows)**: Рекомендуется запускать все последующие Python-команды через специальный терминал **Anaconda Prompt (miniconda3)**, который появится в меню «Пуск» после установки.

> **Критерий самоконтроля:**
> Откройте терминал Anaconda Prompt и введите:
> ```bash
> conda --version
> ```
> *Ожидаемый результат:* Отображение текущей версии менеджера пакетов conda.

---

### Шаг 3: Создание окружения Python
Мы будем использовать Python версии **3.10.12** для стабильной совместимости с библиотеками ML-Agents.

1. В терминале Anaconda Prompt выполните команду для создания изолированного окружения:
   ```bash
   conda create -n mlagents python=3.10.12 -y
   ```
2. Активируйте созданное окружение:
   ```bash
   conda activate mlagents
   ```

> **Критерий самоконтроля:**
> В начале строки терминала в круглых скобках должно отображаться имя активного окружения:
> ```bash
> (mlagents) C:\Users\...
> ```

---

### Шаг 4: Установка PyTorch (с поддержкой CUDA или CPU)
Нейросети в ML-Agents обучаются с помощью PyTorch.

*   **Вариант А (Если на ноутбуке есть дискретная видеокарта NVIDIA)**:
    Установите PyTorch с поддержкой CUDA 12.1 для аппаратного ускорения обучения:
    ```bash
    pip3 install torch~=2.2.1 --index-url https://download.pytorch.org/whl/cu121
    ```
*   **Вариант Б (Если видеокарты нет / встроенная графика Intel/AMD / Apple Silicon M-чипы)**:
    Установите CPU-версию PyTorch:
    ```bash
    pip3 install torch~=2.2.1 --index-url https://download.pytorch.org/whl/cpu
    ```

> [!NOTE]
> Если система на Windows выдает ошибку о недостающих системных библиотеках, установите [Microsoft Visual C++ Redistributable](https://learn.microsoft.com/en-US/cpp/windows/latest-supported-vc-redist?view=msvc-170).

> **Критерий самоконтроля:**
> В активном окружении `(mlagents)` запустите интерактивный тест Python:
> ```bash
> python -c "import torch; print('Версия PyTorch:', torch.__version__); print('Доступность GPU (CUDA):', torch.cuda.is_available())"
> ```
> *Ожидаемый результат:* Вывод версии `2.2.1+cu121` (или `2.2.1+cpu`) и статус `True` (для GPU) или `False` (для CPU).

---

### Шаг 5: Разрешение конфликтов зависимостей (grpcio)
При установке ML-Agents часто возникает ошибка сборки C++ расширений библиотеки `grpcio`. Чтобы избежать её, установите предкомпилированную версию пакета из репозитория `conda-forge`:
```bash
conda install "grpcio=1.48.2" -c conda-forge -y
```

> **Критерий самоконтроля:**
> Команда должна завершиться без ошибок, пакет `grpcio` успешно скачан.

---

### Шаг 6: Установка Python-пакета ML-Agents
Установите строго версию **1.1.0** для совместимости с Unity-пакетами:
```bash
python -m pip install mlagents==1.1.0
```

> [!IMPORTANT]
> **Устранение неполадок с setuptools:**
> В современных версиях Python удален модуль `pkg_resources`. Если запуск выдаст ошибку `ModuleNotFoundError: No module named 'pkg_resources'`, принудительно установите старую версию библиотеки `setuptools`:
> ```bash
> pip install "setuptools<70"
> ```

> **Критерий самоконтроля:**
> Выполните проверку работы консольной утилиты:
> ```bash
> mlagents-learn --help
> ```
> *Ожидаемый результат:* В терминале должен отобразиться крупный текстовый логотип **Unity ML-Agents** и справочная информация.

## Часть 2: Установка Unity 6 и пакетов экосистемы

### Шаг 7: Установка Unity Hub и Unity 6
1. Перейдите на [официальный сайт Unity](https://unity.com/download) и установите **Unity Hub**.
2. Авторизуйтесь под своей учетной записью Unity ID.
3. Перейдите во вкладку **Installs** ➔ кнопка **Install Editor**.
4. В официальных релизах (Official releases) выберите версию **Unity 6000.0 или новее** (Unity 6).
5. При выборе модулей отметьте целевую платформу сборки (например, **Windows Build Support**). Нажмите **Install**.

---

### Шаг 8: Создание базового 3D-проекта
1. В Unity Hub перейдите в раздел **Projects** ➔ кнопка **New project**.
2. Убедитесь, что в качестве редактора выбран установленный **Unity 6**.
3. Выберите шаблон **3D (URP)** (Universal Render Pipeline).
4. Задайте имя проекта (например, `YandexCamp_Robotics`), выберите директорию и нажмите **Create project**.

> **Критерий самоконтроля:**
> Редактор Unity успешно запустился, открылась пустая сцена, во вкладке **Console** нет красных ошибок компиляции.

---

### Шаг 9: Интеграция ML-Agents в Unity
1. В верхнем меню Unity откройте **Window** ➔ **Package Manager**.
2. Нажмите на значок **"+"** в левом верхнем углу окна Package Manager.
3. Выберите пункт **"Add package by name..."**.
4. В поле ввода пропишите следующий идентификатор пакета:
   ```text
   com.unity.ml-agents
   ```
5. Нажмите кнопку **Add** и дождитесь окончания установки.

> [!TIP]
> Если пакет не находится по имени, нажмите на значок шестеренки в Package Manager ➔ Project Settings ➔ включите галочку **Enable Preview Packages** во вкладке Advanced.

> **Критерий самоконтроля:**
> В Package Manager при переключении фильтра на "In Project" отображается пункт **ML-Agents** со статусом установленного пакета.

---

### Шаг 10: Установка ROS-TCP-Connector (Unity Robotics Hub)
Этот пакет отвечает за отправку команд и получение сенсорного потока от ROS-нод робота по TCP-мосту.

1. В окне Package Manager нажмите **"+"** ➔ **"Add package from git URL..."**.
2. Вставьте Git-ссылку на репозиторий с суффиксом пути к пакету:
   ```text
   https://github.com/Unity-Technologies/ROS-TCP-Connector.git?path=/com.unity.robotics.ros-tcp-connector
   ```
3. Нажмите кнопку **Add**.

> **Критерий самоконтроля:**
> В главной верхней панели инструментов среды Unity добавится специализированная вкладка с названием **Robotics**.

---

### Шаг 11: Установка URDF-Importer
Пакет для импорта 3D-моделей роботов (описания сочленений, коллизий и физических параметров) напрямую из файлов URDF.

1. В окне Package Manager нажмите **"+"** ➔ **"Add package from git URL..."**.
2. Вставьте ссылку на репозиторий:
   ```text
   https://github.com/Unity-Technologies/URDF-Importer.git?path=/com.unity.robotics.urdf-importer
   ```
3. Нажмите кнопку **Add**.

> **Критерий самоконтроля:**
> В контекстном меню окна Hierarchy (при клике правой кнопкой мыши) появится пункт **URDF Import**, позволяющий загружать описания роботов.

## Часть 3: Сквозная проверка связи

### Шаг 12: Тест локального соединения
Проверим работоспособность связки Python-скрипта ML-Agents и среды Unity.

1. Запустите приложение **Anaconda Prompt**.
2. Активируйте ваше окружение:
   ```bash
   conda activate mlagents
   ```
3. Запустите тестовое прослушивание симулятора:
   ```bash
   mlagents-learn --run-id=test_connection
   ```
4. **Критерий успешного запуска в консоли:** В окне терминала отобразится строка:
   > *"Listening on port 5004. Start training by pressing the Play button in the Unity Editor."*
5. Перейдите в запущенный проект Unity 6 и нажмите центральную верхнюю кнопку **Play** (запуск сцены).

> **Финальный критерий самоконтроля:**
> В консоли Anaconda Prompt должен начаться динамический обмен пакетами данных. Ошибки отсутствуют, процесс пошел. Поздравляем, ваше рабочее место готово к работе!

# ☁️ Руководство: Запуск обучения Unity ML-Agents в Yandex Cloud (на CPU)

Этот гайд написан для студентов и разработчиков, которые впервые запускают обучение моделей Reinforcement Learning в облаке Yandex Cloud. В нем собраны все шаги — от создания сервера до скачивания готовой модели, а также решения проблем, с которыми вы гарантированно столкнетесь при настройке.

---

## 📋 Обозначения (что нужно заменить на свои данные)
Перед выполнением команд обратите внимание на следующие плейсхолдеры в инструкциях:
*   `<IP_АДРЕС_ВМ>` — публичный IP-адрес вашей виртуальной машины в Yandex Cloud (например, `158.160.14.248`).
*   `<ЛОГИН_ВМ>` — логин пользователя, который вы указали при создании ВМ (например, `yacamptest`).
*   `<ИМЯ_ЮЗЕРА_WINDOWS>` — имя вашей учетной записи на локальном компьютере (можно посмотреть в пути `C:\Users\`).
*   `<ИМЯ_ФАЙЛА_СИМУЛЯТОРА>` — точное имя исполняемого файла вашего симулятора (например, `ROStest.x86_64`).

## Шаг 1: Создание ВМ в Yandex Cloud

Обучение наших моделей происходит на **процессоре (CPU)**. Поскольку нейросеть очень маленькая, видеокарта (GPU) простаивает 95% времени, поэтому мы используем мощную CPU-only конфигурацию.

При создании виртуальной машины (ВМ) в консоли Yandex Cloud укажите следующие параметры:
*   **ОС**: `Ubuntu 24.04 LTS` (или `22.04 LTS`).
*   **Платформа**: `Intel Ice Lake` (или `Cascade Lake`).
*   **Гарантированная доля vCPU**: `100%` (обязательно для стабильной симуляции физики!).
*   **Количество ядер (vCPU)**: `16` (для параллельного запуска 16 сред).
*   **Оперативная память (RAM)**: `32 ГБ`.
*   **Диск**: `50 ГБ` или `100 ГБ`, тип **`Сетевой SSD`** (для быстрого сохранения чекпоинтов).
*   **Сеть**: Включите **«Публичный IP-адрес»** (автоматически).
*   **Прерываемая ВМ (Spot)**: Если хотите сэкономить ~60% бюджета (актуально для тестов), поставьте галочку *«Прерываемая»*. На занятиях лучше её **выключить**, чтобы сервер случайно не остановился во время долгой тренировки.
*   **Логин (Имя пользователя)**: Придумайте логин, например **`yacamptest`** (запомните его, это ваш `<ЛОГИН_ВМ>`).
*   **Доступ**: Выберите *«Сгенерировать пару ключей»*.

## Шаг 2: Настройка SSH-ключей на Windows

При создании ВМ браузер автоматически скачает ZIP-архив с парой ключей. Настроим их:

1.  Создайте на вашем компьютере папку `.ssh` внутри папки вашего пользователя. Путь должен выглядеть так: `C:\Users\<ИМЯ_ЮЗЕРА_WINDOWS>\.ssh\`.
2.  Распакуйте файлы из скачанного ZIP-архива прямо в эту папку.
3.  Там будут два файла: `private_key` (приватный) и `public_key.pub` (публичный). 
4.  Переименуйте файл **`private_key`** (тот, что без расширения) в **`yc_key`**.

> [!WARNING]
> Никогда никому не передавайте приватный ключ `yc_key`. Это ваш пароль для доступа к серверу.

## Шаг 3: Сборка симулятора для Linux Server

Чтобы симулятор работал на облачном сервере без монитора и видеокарты, соберите его в специальном серверном режиме:

1.  Откройте проект в **Unity Editor**.
2.  Перейдите в меню **File ➔ Build Settings** (или **Build Profiles** в Unity 6).
3.  Выберите платформу **Dedicated Server** (или **Linux** и поставьте галочку **Server Build / Dedicated Server**).
    *   *Если платформы Linux нет в списке, установите модуль `Linux Build Support` через Unity Hub.*
4.  В списке сцен выберите вашу рабочую сцену (например, `Scenes/SampleScene`).
5.  **Важно**: Снимите галочку с **`Development Build`** (это ускорит работу симулятора в релизе).
6.  Если активна другая платформа, нажмите **Switch Platform** внизу справа и дождитесь окончания импорта.
7.  Нажмите **Build**, создайте пустую папку (например, `Build_Linux`) и выберите её в качестве пути сохранения.
8.  Зайдите в папку `Build_Linux`, выделите все файлы и заархивируйте их в один ZIP-архив: **`Build_Linux.zip`**.

## Шаг 4: Перенос файлов на сервер

Вам нужно скопировать архив билда и файл конфигурации обучения (`config.yaml`) на сервер.

1.  Откройте программу **PowerShell** на вашем локальном компьютере.
2.  Перейдите в папку вашего Unity-проекта (или туда, где сохранены архив и конфиг):
    ```powershell
    cd C:\path\to\your\UnityProject
    ```
3.  Выполните команды безопасного копирования (`scp`), указав путь к ключу, IP-адрес вашей ВМ и ваш логин:

```powershell
# Копируем архив симулятора
scp -i C:\Users\<ИМЯ_ЮЗЕРА_WINDOWS>\.ssh\yc_key Build_Linux.zip <ЛОГИН_ВМ>@<IP_АДРЕС_ВМ>:~/

# Копируем конфиг обучения
scp -i C:\Users\<ИМЯ_ЮЗЕРА_WINDOWS>\.ssh\yc_key config.yaml <ЛОГИН_ВМ>@<IP_АДРЕС_ВМ>:~/
```

## Шаг 5: Настройка Linux-окружения на сервере

1.  Подключитесь к серверу по SSH из PowerShell вашего компьютера:
    ```powershell
    ssh -i C:\Users\<ИМЯ_ЮЗЕРА_WINDOWS>\.ssh\yc_key <ЛОГИН_ВМ>@<IP_АДРЕС_ВМ>
    ```
2.  После успешного входа выполните команды настройки по порядку.

### 1. Установка unzip и системных библиотек
```bash
sudo apt-get update
sudo apt-get install -y unzip wget git build-essential
```

### 2. Установка Anaconda (Miniconda)
```bash
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
bash Miniconda3-latest-Linux-x86_64.sh -b -p $HOME/miniconda

# Интегрируем conda в командную строку
source $HOME/miniconda/bin/activate
conda init
source ~/.bashrc
```

### 3. Принятие Лицензионного соглашения Anaconda
> [!IMPORTANT]
> **ПОДВОДНЫЙ КАМЕНЬ №1**: Новые версии Conda блокируют установку пакетов, пока вы не примете соглашение. Выполните эти команды:
```bash
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/main
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/r
```

### 4. Создание окружения Python и установка PyTorch (CPU-версии)
```bash
conda create -n mlagents python=3.10.12 -y
conda activate mlagents

# Ставим легкую версию PyTorch для процессоров (без CUDA)
pip install torch~=2.2.1 --index-url https://download.pytorch.org/whl/cpu
```

### 5. Установка ML-Agents и исправление несовместимости версий
> [!IMPORTANT]
> **ПОДВОДНЫЙ КАМЕНЬ №2**: В современных версиях Python удален модуль `pkg_resources`. Если просто поставить ML-Agents, он выдаст ошибку `ModuleNotFoundError: No module named 'pkg_resources'`.
> Чтобы это исправить, нужно принудительно установить старую версию библиотеки `setuptools` (версию ниже 70-й):

```bash
# Решаем конфликты grpcio
conda install "grpcio=1.48.2" -c conda-forge -y

# Ставим совместимый setuptools (обязательно в кавычках!)
pip install "setuptools<70"

# Ставим пакет ML-Agents
python -m pip install mlagents==1.1.0
```

Проверяем работоспособность:
```bash
mlagents-learn --help
```
*(Должен отобразиться логотип ML-Agents и справка).*

## Шаг 6: Запуск обучения

1.  Распакуйте архив с симулятором:
    ```bash
    unzip ~/Build_Linux.zip -d ~/Build_Linux
    ```
2.  Посмотрите, как называется папка и исполняемый файл внутри:
    ```bash
    ls ~/Build_Linux/Build_Linux
    # (Вы увидите файлы, например, ROStest.x86_64 and ROStest_Data)
    ```
3.  Дайте права на выполнение файлу симулятора (замените `<ИМЯ_ФАЙЛА_СИМУЛЯТОРА>` на ваше имя файла, например `ROStest.x86_64`):
    ```bash
    chmod +x ~/Build_Linux/Build_Linux/<ИМЯ_ФАЙЛА_СИМУЛЯТОРА>
    ```
4.  Запустите обучение:

> [!IMPORTANT]
> **ПОДВОДНЫЙ КАМЕНЬ №3**: 
> *   Используйте только **абсолютные пути** (начинающиеся с `/home/...`), символ тильды `~` внутри ML-Agents может не распознаться, что приведет к ошибке *«Provided filename does not match any environments»*.
> *   Передавайте параметр `-logFile /dev/null` после `--env-args`, чтобы отключить запись логов симулятора на диск (иначе они забьют всю память).

```bash
# Единая команда для запуска (замените пути и <ИМЯ_ФАЙЛА_СИМУЛЯТОРА> на свои)
mlagents-learn /home/<ЛОГИН_ВМ>/config.yaml --run-id=cloud_cpu_test --env=/home/<ЛОГИН_ВМ>/Build_Linux/Build_Linux/<ИМЯ_ФАЙЛА_СИМУЛЯТОРА> --num-envs=16 --no-graphics --env-args -logFile /dev/null
```

## Шаг 7: Скачивание обученной модели

После окончания обучения (или если вы прервали его клавишами `Ctrl + C`), готовый файл обученной нейросети (`.onnx`) будет сохранен в папку результатов.

Чтобы забрать его на свой ПК, откройте **PowerShell локального компьютера** (не на сервере!) и выполните команду:

```powershell
# Скачивание финальной модели прямо в папку Assets вашего проекта Unity
scp -i C:\Users\<ИМЯ_ЮЗЕРА_WINDOWS>\.ssh\yc_key <ЛОГИН_ВМ>@<IP_АДРЕС_ВМ>:~/results/cloud_cpu_test/GFSX_Brain.onnx C:\path\to\your\UnityProject\Assets\
```

Если вы хотите забрать промежуточный чекпоинт:
```powershell
scp -i C:\Users\<ИМЯ_ЮЗЕРА_WINDOWS>\.ssh\yc_key <ЛОГИН_ВМ>@<IP_АДРЕС_ВМ>:~/results/cloud_cpu_test/GFSX_Brain/GFSX_Brain-<КОЛИЧЕСТВО_ШАГОВ>.onnx C:\path\to\your\UnityProject\Assets\
```

## Шаг 8: Просмотр графиков в TensorBoard (SSH-туннель)

Вы можете отслеживать графики вознаграждений (Reward) и сходимости обучения прямо в процессе тренировки через ваш обычный браузер:

1.  **На сервере** запустите TensorBoard (не выключая обучение, в соседней SSH-сессии):
    ```bash
    tensorboard --logdir ~/results --port 6006
    ```
2.  **На локальном компьютере** в новом окне PowerShell пробросьте порт с помощью SSH-туннелирования:
    ```powershell
    ssh -i C:\Users\<ИМЯ_ЮЗЕРА_WINDOWS>\.ssh\yc_key -L 6006:127.0.0.1:6006 <ЛОГИН_ВМ>@<IP_АДРЕС_ВМ>
    ```
3.  Откройте браузер на вашем ПК и перейдите по адресу: **`http://localhost:6006`**

Вы увидите графики обучения, обновляющиеся в реальном времени!

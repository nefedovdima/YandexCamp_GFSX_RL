# 📖 Пояснения к `config.yaml`

Справочник по всем параметрам файла обучения `config.yaml` проекта GFS-X.
Файл состоит из двух частей:

1. **`behaviors:`** — гиперпараметры алгоритма PPO (как учится нейросеть).
2. **`environment_parameters:`** — настраиваемые параметры симуляции, которые
   `RobotBrain.cs` читает в начале каждого эпизода.

> ⚠️ **Важно про пересборку.**
> Меняя **значения в `environment_parameters`**, билд пересобирать НЕ нужно —
> правишь yaml и перезапускаешь `mlagents-learn`. А вот меняя **сам код скриптов**
> (например, добавляя новый параметр или новую награду) — билд надо пересобрать один раз.
>
> ⚠️ **Только латиница.** Внутри `config.yaml` не должно быть кириллицы и кириллических
> комментариев — PyYAML упадёт с ошибкой парсинга. Все пояснения держим в этом файле.
>
> ⚠️ **`config.yaml` теперь без комментариев.** Файл держим чистым: только ключи и значения,
> блоки разделены пустой строкой. Пояснения к наградам вынесены в отдельный файл
> **`REWARDS_REFERENCE.md`** (за что отвечает каждый параметр награды/штрафа), общий
> справочник — здесь.

## Часть 1. `behaviors → GFSX_Brain` — гиперпараметры PPO

### Общее
| Параметр | Значение | Что делает |
| :--- | :--- | :--- |
| `trainer_type` | `ppo` | Алгоритм обучения — Proximal Policy Optimization. |
| `max_steps` | `5000000` | Сколько всего шагов симуляции пройти и остановиться. Значение из методички P3; для полноценного обучения — 5–10 млн. **Влияет на `learning_rate_schedule: linear`**: LR падает к нулю именно к `max_steps`, поэтому последние ~10% шагов почти не учат. |
| `time_horizon` | `1000` | Сколько шагов опыта собрать с агента до подсчёта отдачи (для длинных задач — больше). |
| `summary_freq` | `5000` | Как часто (в шагах) писать статистику в TensorBoard и в консоль (в т.ч. метрики `Attempt/*` и `Rewards/*` из `RobotStats`). |
| `keep_checkpoints` | `5` | Сколько последних чекпоинтов `.onnx` хранить на диске. |

### `hyperparameters` — как обновляется нейросеть
| Параметр | Значение | Что делает | Куда крутить |
| :--- | :--- | :--- | :--- |
| `batch_size` | `2048` | Сколько шагов опыта берётся в один шаг градиентного спуска. | Больше → стабильнее, но медленнее. |
| `buffer_size` | `40960` | Сколько опыта накопить перед обновлением весов. Должен быть кратен `batch_size`. | Больше → устойчивее, дольше сбор. |
| `learning_rate` | `0.00025` | Скорость обучения (размер шага). | Слишком большой → «разваливается»; слишком малый → учится вечно. |
| `beta` | `0.005` | Сила энтропии — «любопытство»/случайность действий, помогает исследовать арену. | Робот застревает в одной стратегии → повысить; хаотичен → понизить. |
| `epsilon` | `0.2` | Ограничение шага PPO (насколько политика может измениться за обновление). | Обычно не трогают. |
| `lambd` | `0.95` | GAE-параметр: баланс между смещением и дисперсией оценки отдачи. | Обычно 0.9–0.95. |
| `num_epoch` | `3` | Сколько раз прогнать буфер опыта при каждом обновлении. | Больше → полнее учёба на данных, риск переобучения. |
| `learning_rate_schedule` | `linear` | `linear` — LR плавно падает к нулю к `max_steps`; `constant` — не меняется. | Для дообучения (`--resume`) иногда ставят `constant`. |

### `network_settings` — размер и память сети
| Параметр | Значение | Что делает |
| :--- | :--- | :--- |
| `normalize` | `true` | Приводит вектор наблюдений к единому масштабу (важно, т.к. датчики разнородны). |
| `hidden_units` | `256` | Нейронов в скрытом слое. Больше → «умнее», но медленнее и прожорливее. |
| `num_layers` | `2` | Число скрытых слоёв. |
| `memory.sequence_length` | `64` | Длина последовательности, которую видит LSTM за раз. |
| `memory.memory_size` | `256` | Размер рекуррентной памяти (LSTM). Даёт «память» о недавних событиях — компенсирует потерю мяча из кадра и задержки. |

### `reward_signals → extrinsic`
| Параметр | Значение | Что делает |
| :--- | :--- | :--- |
| `gamma` | `0.99` | Дисконт будущих наград. Ближе к 1 → робот больше ценит отложенную цель (захват), чем сиюминутные бонусы. |
| `strength` | `1.0` | Множитель внешней (заданной нами) награды. |

## Часть 2. `environment_parameters` — параметры симуляции

Эти ключи читаются в `RobotBrain.ApplyTunables()` в начале КАЖДОГО эпизода.
Если ключа нет в yaml — берётся значение из инспектора Unity (fallback).

### 🎯 Веса наград
| Ключ | Значение | За что | Куда крутить |
| :--- | :--- | :--- | :--- |
| `approach_reward` | `1.0` | **Вес потенциала**, а не доход за тик. Близость к мячу; вблизи мяча сближение ценится сильнее, чем вдали (требование P3, шаг 4.1) — это зашито в `DistancePotential()`. Путь 2 м → 0 м даёт суммарно `+2`. | Вяло едет к мячу → повысить. |
| `centering_reward` | `0.5` | **Вес потенциала**: насколько робот повёрнут мордой к мячу. Считается по **истинному пеленгу**, а НЕ по `cam.BallVisible` — иначе на границе кадра потенциал обрывается и робот начинает пятиться, лишь бы вернуть мяч в кадр. | Не доворачивается к мячу → повысить. |
| `action_penalty` | `0.01` | Штраф за **резкость** — изменение газа/руля между шагами (плавность). | Дёргается → повысить; слишком вялый → понизить. |
| `obstacle_penalty` | `0.01` | Штраф за **близость** к преграде по боковому ИК (сближение, ещё не касание). | Подъезжает к стенам вплотную → повысить. |
| `wall_collision_penalty` | `0.05` | Штраф за **факт касания стены** (слои из `Wall Collision Mask`). Контакты с полом отфильтрованы по нормали. | Таранит стены → повысить. |
| `obstacle_collision_penalty` | `0.03` | Штраф за **факт касания преграды/коробки** (слои из `Obstacle Collision Mask`). | Толкает коробки → повысить. |
| `forward_penalty` | `0.0` | Штраф за газ вперёд (`gas>0`). Обычно 0. | Держать 0. |
| `grab_reward` | `5.0` | Крупный терминальный бонус за успешный захват+удержание. | Главная цель; менять осторожно. |
| `fall_penalty` | `-1.0` | Штраф за падение с арены (ниже `fallY`). ⚠️ **Сейчас недостижим**: в сцене у робота `m_Constraints: 84`, куда входит `FreezePositionY` — упасть он физически не может. Либо снять заморозку Y (методичка P2 просит замораживать только вращения X/Z = 80), либо считать этот ключ мёртвым. | — |
| `timeout_penalty` | `-0.3` | Штраф, если время эпизода истекло без захвата. | — |
| `hold_tick_reward` | `0.02` | Награда за каждый тик удержания мяча (стимул держать стабильно). | — |
| `time_penalty` | `0.001` | Штраф за каждый шаг → срочность. | Долго ищет/тупит → повысить (0.002). |
| `spin_penalty` | `0.0` | Штраф за \|поворот\|. **Выключен намеренно.** | Держать на 0, см. предупреждение ниже. |
| `reverse_penalty` | `0.0` | Штраф за газ назад (камера и клешня спереди). | Робот пятится → включить `0.002`. |
| `shaping_gamma` | `0.99` | Гамма в формуле shaping'а. **Обязана совпадать** с `reward_signals.extrinsic.gamma`. | Не трогать без нужды. |

> ⚠️ **Не боритесь с кружением через `spin_penalty` и `centering_reward`.** Это уже пробовали,
> и стало хуже. Кружение — не самоцель агента, а способ **держать мяч в кадре ради ренты**
> `centering_reward`. Поэтому:
> * `spin_penalty` ↑ облагает налогом руль → агент переходит на широкие дуги газом
>   (**едет задом по кругу**) и продолжает фармить то же самое, только дешевле;
> * `centering_reward` ↑ **увеличивает ренту** и делает кружение ещё выгоднее.
>
> Правильный рычаг — геометрия обзора (`field of view` камеры) и форма награды:
> награда считается как `shaping_gamma*Ф(s') - Ф(s)`, поэтому любой замкнутый маршрут
> даёт в сумме ~0 и накрутить её кружением уже нельзя.

### ⏱ Эпизод
| Ключ | Значение | Что делает | Куда крутить |
| :--- | :--- | :--- | :--- |
| `episode_time_limit` | `90.0` | Секунд на одну попытку до таймаута (**1.5 минуты**). | Мало времени на дальние спавны → повысить. |
| `hold_ticks_required` | `50` | Сколько тиков подряд держать мяч для успеха (~1 с при 50 Гц). | Хочешь строже удержание → повысить. |

### 🚗 Движение
| Ключ | Значение | Что делает | Куда крутить |
| :--- | :--- | :--- | :--- |
| `max_linear_cmd` | `0.25` | Жёсткий лимит поступательной скорости, м/с. | Медленно ездит и не успевает → повысить, но синхронизируй с реальным роботом. |

### 🎾 Спавн мяча / сложность поиска
| Ключ | Значение | Что делает | Куда крутить |
| :--- | :--- | :--- | :--- |
| `ball_spawn_jitter` | `0.15` | Случайный сдвиг мяча от стартовой точки (когда не используется случайный спавн по арене), м. | — |
| `ball_spawn_radius` | `0.0` | `0` = мяч по всей арене (**так и задумано**: мяч за коробкой — легальный сценарий, робот должен учиться искать, см. `EnvironmentRandomizer.cs:136`); `>0` = мяч не дальше R метров от робота. | Ставить `>0` только как **временную диагностику** («он вообще способен доехать?») и возвращать в `0`. Постоянно — не надо: политика заточится под лёгкий режим и хуже переносится. |

### 🎲 Domain Randomization
| Ключ | Значение | Что делает | Куда крутить |
| :--- | :--- | :--- | :--- |
| `dr_enabled` | `1.0` | Мастер-выключатель ВСЕХ шумов/задержек/рандомизации. `0` = чистая симуляция. | Отладка поведения → поставить `0`. |
| `us_noise` | `0.05` | Амплитуда белого шума УЗ-дальномера (±доля). | Робот «дёргается» от шума → понизить. |
| `yolo_dropout_chance` | `0.15` | Шанс «ослепить» камеру на резком повороте (потеря мяча). | — |
| `latency_min` | `8` | Мин. задержка команд, тиков (8 = 160 мс). | Синхронизируй с реальным пингом Wi-Fi/ROS. |
| `latency_max` | `13` | Макс. задержка команд, тиков (13 = 260 мс). | — |
| `mass_min` | `1.0` | Мин. масса робота при рандомизации, кг. | — |
| `mass_max` | `4.0` | Макс. масса робота при рандомизации, кг (база 2.5 кг). | — |

> Ещё несколько диапазонов DR (скорости, масса/размер мяча, инерция моторов) вынесены
> в **инспектор Unity** как поля `drMoveSpeedRange`, `drTurnSpeedRange`, `drBallMassMul`
> и т.д. — их удобно править прямо в редакторе. В `environment_parameters` вынесены только
> самые часто крутимые.

## 🔧 Типовые сценарии тюнинга

| Проблема | Что менять |
| :--- | :--- |
| Робот кружит / едет задом вокруг мяча | Проверить оптику: `horizontalFovDeg` в `SimulatedYoloCamera` должен быть **40** (требование P3). Держать `spin_penalty: 0.0`. Не поднимать `centering_reward` — это рента, ради которой он и кружит. |
| Подъезжает, но не хватает мяч | **Слепая зона перед захватом — это норма, а не баг.** P3 прямо пишет: «мяч скрывается под бампером перед захватом», и проезжать её должна LSTM-память (`memory_size: 256`). Не убирайте зону наклоном камеры. Убедитесь, что награда не штрафует уход в слепоту: `centering_reward` обязан считаться по пеленгу, а не по `cam.BallVisible`. |
| Не находит мяч (теряет из виду) | Проверить `horizontalFovDeg: 40` и `maxRange: 2.0`. Мяч дальше 2 м не детектируется в принципе — это ожидаемо, робот должен искать. |
| Дёргается, едет рывками | `action_penalty` ↑ |
| Врезается в стены | `obstacle_penalty` ↑ |
| Не успевает доехать за эпизод | `episode_time_limit` ↑ или `max_linear_cmd` ↑ |
| Хочу отладить «чистое» поведение без шумов | `dr_enabled: 0.0` |
| Обучение «разваливается» (reward падает в минус) | `learning_rate` ↓, `beta` ↓ |
| Робот застрял в одной стратегии | `beta` ↑ |

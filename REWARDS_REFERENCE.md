# 🎯 Справочник наград и штрафов GFS-X

Отдельный файл — что делает **каждый** параметр награды/штрафа из блока
`environment_parameters` в `config.yaml`. Сам `config.yaml` держим без комментариев
(чистый YAML, блоки разделены пустой строкой) — все пояснения здесь.

> Значения из `environment_parameters` подхватываются `RobotBrain.cs` в начале КАЖДОГО
> эпизода (метод `ApplyTunables()`), поэтому менять их можно **без пересборки билда** —
> правишь yaml и перезапускаешь `mlagents-learn`. Если ключа в yaml нет — берётся значение
> из инспектора Unity. В визуальном запуске (без тренера) yaml не читается — там работают
> значения из инспектора.

## ⛔ Curriculum отменён — решили идти классическим путём

Ранее здесь была пошаговая лестница (Stage 0 → 7: sanity check → полная арена → узкий DR →
широкий DR → задержка → шумы → одометрия → поиск). Stage 0 и Stage 1 показали отличный
результат (Grabs 0.04→0.98!), но переход на Stage 2 оказался хрупким к процессным багам
(prefab/билд рассинхрон) и съедал непропорционально много времени на калибровку каждого шага.

**Решение:** отказаться от пошагового наращивания сложности. Вместо этого — **классический
подход**: сразу вся динамическая рандомизация (масса, скорость, освещение, задержка команд,
шум датчиков, пропадание кадров камеры, одометрия) + аккуратно подобранные раз и навсегда
величины наград, без калибровки по этапам. Ниже — финальная конфигурация и экономика наград.

---

## Как устроена награда (важно понять до кручения чисел)

Награда за шаг складывается из нескольких слагаемых. Ключевое — **potential-based
shaping**: награда за приближение считается как `shaping_gamma * Ф(s') − Ф(s)`, где `Ф(s)`
(«потенциал») тем больше, чем ближе робот к мячу и чем точнее он повёрнут к нему мордой.
Это математически гарантирует (теорема Ng et al., 1999), что **накрутить награду
кружением/«подъехал-отъехал» нельзя**: сумма по любому замкнутому маршруту ≈ 0, платит
только фактический прогресс к цели.

Поэтому `approach_reward` и `centering_reward` — это **веса потенциала**, а НЕ доход за
тик. Остальные параметры — прямые бонусы/штрафы за тик или за событие.

> ⚠️ **Протечка shaping на длинных эпизодах.** При `γ<1` shaping даёт паразитный «доход»
> ≈ `(1−γ)·|Ф|` за каждый шаг, пока `Ф` отрицательный (робот далеко от мяча). Гасится
> `time_penalty`: при 90 с (4500 тиков) и `time_penalty=0.01` полный тайм-аут стоит ≈ −45,
> этого достаточно, чтобы «дожить до конца» было хуже, чем рискнуть и попытаться схватить.

### 💰 Экономика на 90-секундном эпизоде (грубая прикидка)

| Сценарий | Итог |
| :--- | :--- |
| Полный тайм-аут (ничего не поймал) | `-0.01×4500 - 0.5 ≈ -45.5` |
| Быстрый захват за ~20 с (1000 тиков) | `-0.01×1000 + grab(12) + hold(2.5) ≈ +4.5` (без учёта shaping-бонуса за сближение, который добавит ещё) |
| Зависнуть в луче клешни, не хватая | `gripper_ir(0.005) - time_penalty(0.01) = -0.005/тик` — **чистый минус**, зависать невыгодно |

Захват явно выгоднее любого другого исхода, но не настолько доминирует, чтобы обесценить
штрафы за столкновения — несколько ударов в стену (`0.15` каждый) заметно откусывают от
приза за захват (`12.0`), а не тонут в нём незаметно.

## 🎾 Целевые бонусы

| Ключ | Значение | За что отвечает | Куда крутить |
| :--- | :--- | :--- | :--- |
| `approach_reward` | `1.0` | Вес потенциала «близость к мячу». Подъезд с 2 м вплотную даёт суммарно ≈ `+2`; вблизи мяча сближение ценится сильнее, чем вдали. | Вяло едет к мячу → повысить. |
| `centering_reward` | `0.3` | Вес потенциала «робот повёрнут мордой к мячу / держит мяч в поле зрения камеры». Считается по **истинному пеленгу**, а не по видимости в кадре (иначе на границе кадра робот начинает пятиться). Держим умеренным, чтобы не конкурировать с `time_penalty`/срочностью. | Не доворачивается к мячу → повысить немного. |
| `grab_reward` | `12.0` | Терминальный бонус за успешный захват + удержание `hold_ticks_required` тиков. Подобран **умеренным**: достаточно большой, чтобы оставаться приоритетной целью (перевешивает тайм-аут ≈−45), но не настолько огромный, чтобы штрафы за столкновения/резкость становились незаметны на его фоне. | Мало захватов, штрафы не мешают → можно повысить; робот таранит стены ради скорости → понизить. |
| `hold_tick_reward` | `0.05` | Награда за каждый тик, пока мяч удерживается в клешне (стимул держать стабильно до успеха). | Мяч роняет сразу после захвата → повысить. |
| `gripper_ir_reward` | `0.005` | Награда за тик, пока мяч уже в луче клешни, но ещё не схвачен («последний метр»). **Обязана быть меньше `time_penalty`**, иначе зависать в луче без захвата станет выгоднее, чем закрыть клешню (это реально ломало обучение — держать строго ниже). | Не трогать без явной проверки экономики. |
| `grab_miss_penalty` | `0.0` | Штраф за команду «закрыть» вхолостую (мяча в клешне нет). Выключен по умолчанию. | Робот спамит захватом впустую → включить `0.01`. |
| `search_find_reward` | `0.5` | **Разовый** бонус за первое обнаружение мяча в эпизоде камерой (переход «не виден → виден»). Несколько баллов — ключевой сигнал для обучения поиску скрытого мяча, но не самоцель. Даётся один раз, накрутить нельзя. | Не учится искать → повысить (до 1.0). |
| `ball_visible_reward` | `0.005` | Награда за **каждый тик**, пока мяч виден в камере. Единственный прямой стимул для сервопривода камеры активно следить за мячом (сам поворот камеры — отдельное действие, раньше поощрялся только косвенно). **Обязана быть меньше `time_penalty`**, иначе выгодно замереть с мячом в кадре вместо захвата (та же логика защиты, что у `gripper_ir_reward`). | Камера не следит за мячом, теряет цель → повысить чуть (0.008), не выше `time_penalty`. |

## ⚠️ Терминальные штрафы

| Ключ | Значение | За что отвечает | Куда крутить |
| :--- | :--- | :--- | :--- |
| `fall_penalty` | `-1.0` | Штраф за падение с арены. ⚠️ В текущей сцене у робота заморожен `FreezePositionY` — физически недостижим. | — |
| `timeout_penalty` | `-0.5` | Терминальный штраф при истечении `episode_time_limit` без захвата (сверх накопленного `time_penalty` за все тики). | — |

## 🚦 Пошаговые штрафы (каждый тик)

| Ключ | Значение | За что отвечает | Куда крутить |
| :--- | :--- | :--- | :--- |
| `time_penalty` | `0.01` | Константный штраф за **каждый тик** — не очень большой, но создаёт стремление завершить попытку досрочно (успехом, а не тайм-аутом) и гасит протечку shaping. | Тянет время/кружит → повысить (0.015). |
| `action_penalty` | `0.02` | Штраф за **резкость** управления: модуль изменения газа, руля **и камеры** между шагами в одной формуле. Одного параметра достаточно — отдельные штрафы за реверс газа и за движение камеры были избыточны (дублировали этот же сигнал) и убраны для простоты. | Дёргается/рывками (в т.ч. камерой) → повысить. |
| `obstacle_penalty` | `0.01` | Штраф за **близость** к преграде по боковым ИК (сближение, ещё не касание). | Подъезжает вплотную, но не бьётся → повысить. |

## 💥 Штрафы за столкновения (факт касания) — «достаточно немалые»

Отличаются от `obstacle_penalty`: тот про *близость* по ИК, эти — про *фактический контакт*
коллайдеров. Пол на слое Obstacle, поэтому засчитываются только контакты с почти
горизонтальной нормалью (стены/бока коробок), контакт с полом игнорируется.

| Ключ | Значение | За что отвечает | Куда крутить |
| :--- | :--- | :--- | :--- |
| `wall_collision_penalty` | `0.15` | Штраф за тик касания **стены**. Поднят заметно (было 0.05) — таран стен должен ощутимо откусывать от приза за захват (12.0), а не быть косметикой. | Всё ещё таранит → повысить (0.2). |
| `obstacle_collision_penalty` | `0.08` | Штраф за тик касания **преграды/коробки**. Немного мягче стен — коробки менее «фатальны». | Толкает коробки → повысить. |

> Разделение стен и коробок по слоям + `Wall/Obstacle Collision Mask` в `RobotBrain` — см.
> `WORK_REPORT_2.md`.

## ↕️ Штрафы за направление движения

| Ключ | Значение | За что отвечает | Куда крутить |
| :--- | :--- | :--- | :--- |
| `spin_penalty` | `0.0` | Штраф за `\|руль\|`. Держать 0: при узком FOV камеры робот обязан вращаться, чтобы искать мяч. | Не трогать. |
| `forward_penalty` | `0.0` | Штраф за газ вперёд. Не нужен. | Держать 0. |

| Ключ | Значение | Кратко |
| :--- | :--- | :--- |
| `episode_time_limit` | `90.0` | Секунд на попытку. Полное время по методичке; экономика наград подобрана под это значение (см. таблицу выше). |
| `hold_ticks_required` | `50` | Тиков подряд держать мяч для успеха (~1 с при 50 Гц). |

| `ball_spawn_jitter` | `0.15` | Запасной сдвиг мяча, если случайная точка не нашлась. |
| `ball_spawn_radius` | `0.0` | `0` = мяч по всей арене (полная сложность поиска). |
| `hidden_spawn_fraction` | `0.3` | Доля эпизодов, где мяч **гарантированно не виден со старта** (за стеной/коробкой). Умеренное значение: сохраняет цель «учить искать», но не топит обучение (0.5–1.0 давали слишком глубокую просадку). |
| `box_count_min` / `box_count_max` | `4` / `15` | Число коробок-преград за эпизод. |

| `us_noise` | `0.05` | Белый шум УЗ-дальномера. |
| `odom_drift` | `0.03` | Накопительный дрейф одометрии (obs 11/12/13), имитирует уход энкодеров/IMU реального робота. |
| `odom_noise` | `0.01` | Белый шум одометрии за тик. |
| `yolo_dropout_chance` | `0.15` | Шанс «ослепить» камеру (пропадание кадров) при резком повороте. |
| `latency_min` / `latency_max` | `8` / `13` | Задержка команд, тиков (160–260 мс — реальный Wi-Fi/ROS пинг). |
| `mass_min` / `mass_max` | `1.0` / `4.0` | Разброс массы робота, кг (база 2.5). |

## 📊 Быстрые рецепты

| Симптом | Что крутить |
| :--- | :--- |
| Врезается в стены | `wall_collision_penalty` ↑ |
| Толкает/двигает коробки | `obstacle_collision_penalty` ↑ |
| Подъезжает вплотную, но не таранит | `obstacle_penalty` ↑ |
| Дёргается (в т.ч. камерой) | `action_penalty` ↑ |
| Долго тупит, тянет время | `time_penalty` ↑ |
| Не учится искать скрытый мяч | `search_find_reward` ↑, либо `hidden_spawn_fraction` ↓ (было слишком сложно) |
| Зависает у мяча, не хватая | проверь `gripper_ir_reward < time_penalty` |
| Не успевает за попытку | `episode_time_limit` ↑ или `max_linear_cmd` ↑ |
| Роняет мяч сразу после захвата | `hold_tick_reward` ↑ |
